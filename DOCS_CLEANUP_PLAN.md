# Docs Cleanup & Missing Content Plan

## Current issues

### 1. node_modules leaking into docs (again)
`docs/ui-site/ui/node_modules/` and `docs/ui-site/json-forms/*/node_modules/` contain hundreds of
third-party README files. VuePress picks them up despite `!**/node_modules` in `pagePatterns`
because the `copyReadme` script copies full library trees without pruning nested node_modules.

**Fix:** Add `node_modules` to the `EXCLUDE_ALWAYS` list in `tools/doc/vuepress.mjs` so `copyReadme`
never descends into them. Then delete the leaked directories.

---

### 2. Malformed files at docs/ root
`docs/core-siteREADME.md` and `docs/ui-siteREADME.md` exist (missing `/` in path).
These are caused by `copyReadme('', 'core-site', 1)` — the root-level README.md is copied as
`docs/core-site` + `README.md` but the join produces `docs/core-siteREADME.md`.

**Fix:** In `copyReadme`, use `path.join(toDir, path.relative(from, file))` correctly, or just
copy the root README manually to both sites as `docs/ui-site/README.md` and `docs/core-site/README.md`.
Delete the two malformed files.

---

### 3. Missing parent README.md files (section landing pages)

The following sidebar sections have sub-pages but no parent `README.md`, so clicking the
section heading in VuePress leads nowhere:

| Section | Missing file | Sub-pages present |
|---|---|---|
| core-site › health | `docs/core-site/health/README.md` | api/ |
| core-site › logging | `docs/core-site/logging/README.md` | api/, vue/ |
| core-site › tools | `docs/core-site/tools/README.md` | api/, vue/, authentication/ |
| ui-site › json-forms | `docs/ui-site/json-forms/README.md` | api/, core/, vue/, vue-e2e/ |

**Fix:** Create a short introductory README.md for each of these sections.

---

### 4. Stub content (Nx placeholder text)

The following files were auto-generated by Nx and contain only "This library was generated with Nx":

- `docs/core-site/health/api/README.md`
- `docs/core-site/logging/api/README.md`

Health and logging are sub-modules of `libs/tools/api/` (see `libs/tools/api/src/lib/health/`
and `libs/tools/api/src/lib/logging/`). The content in `docs/core-site/tools/api/README.md` is
better and already covers health and logging usage.

**Fix:** Replace stubs with meaningful content referencing the tools-api docs, or redirect to
`tools/api`.

---

### 5. No docs for `libs/core/types`

`libs/core/types/` exists with 7 TypeScript files but has no docs section in core-site.

**Fix:** Add `copyReadme('libs/core', 'core-site/core')` + `createMenu('core-site/core')` in
`vuepress.mjs`, add a sidebar entry in `core-site/.vuepress/config.js`, and write a README.md
in `libs/core/types/`.

---

### 6. `docs/data/` directory at docs/ root

An empty `docs/data/` directory exists. It is not part of any site.

**Fix:** Delete or repurpose — if it serves as shared data/fixtures for VuePress demos, document
its purpose; otherwise remove it.

---
